## Introducción 
El ciclo de vida del desarrollo de software (Software Development Life Cycle - SDLC) define un conjunto estructurado de fases que permiten planificar, diseñar, construir, probar, implementar y mantener sistemas de software de manera organizada. Los estándares internacionales de ingeniería de software como ISO/IEC 12207, ISO/IEC 15288 y diversas normas del IEEE establecen buenas prácticas para la documentación y los entregables que deben generarse en cada etapa del proceso.

---

## Fase de Planificación

En esta fase se define la visión general del proyecto, se analizan los recursos necesarios y se determina la viabilidad del desarrollo del sistema.

### Documentos y Entregables 
- Acta de inicio del proyecto (Project Charter)
- Documento de visión del sistema
- Estudio de factibilidad (técnica, económica y operativa)
- Identificación de stakeholders
- Plan de gestión del proyecto
- Plan de gestión de riesgos
- Cronograma del proyecto (Matriz RACI o diagrama de Gantt)
- Estimación de costos y recursos
- Plan de comunicación (Metodología ágil)
- Plan de calidad del proyecto

---

## Fase de Análisis de Requerimientos

El objetivo de esta fase es comprender completamente el problema que se desea resolver y documentar las funcionalidades y restricciones del sistema.

### Documentos y Entregables
- Especificación de Requerimientos del Software (SRS)
- Listado de requerimientos funcionales
- Listado de requerimientos no funcionales
- Casos de uso del sistema
- Historias de usuario (en metodologías ágiles)
- Diagramas de procesos o flujos
- Modelo conceptual del dominio
- Prototipos o mockups del sistema
- Matriz de trazabilidad de requerimientos

---

## Fase de Diseño del Sistema

En esta etapa se define la arquitectura del sistema y se establecen las estructuras técnicas que permitirán desarrollar el software.

### Documentos y Entregables
- Documento de Diseño del Software (SDD)
- Arquitectura del sistema
- Diagramas UML (clases, secuencia, componentes y despliegue)
- Modelo de base de datos y diagrama entidad-relación
- Diseño de interfaces de usuario (UI/UX)
- Definición de APIs
- Diseño de seguridad del sistema

---

## Fase de Codificación

En esta fase se desarrolla el sistema de software a partir de los diseños definidos en la etapa anterior.

### Entregables
- Código fuente del sistema
- Repositorio de código (Git u otras plataformas)
- Documentación del código
- Manual técnico del desarrollador
- Scripts de base de datos
- Configuración de entornos
- Versionamiento del software

---

## Fase de Pruebas

El objetivo de esta fase es verificar que el sistema funcione correctamente y cumpla con los requerimientos definidos.

### Documentos y Entregables
- Plan de pruebas
- Casos de prueba
- Datos de prueba
- Scripts de pruebas automatizadas
- Informe de resultados de pruebas
- Reporte de defectos (bugs)
- Pruebas unitarias
- Pruebas de integración
- Pruebas de sistema
- Pruebas de aceptación del usuario

---

## Fase de Implementación

En esta fase el sistema se despliega en el entorno real de producción.

### Documentos y Entregables
- Plan de despliegue
- Manual de instalación
- Manual de usuario
- Manual de operación
- Configuración del entorno de producción
- Plan de capacitación
- Notas de versión (Release Notes)
- Plan de reversión o rollback

---

## Fase de Mantenimiento

El mantenimiento del software permite corregir errores, mejorar el sistema y adaptarlo a nuevas necesidades.

### Tipos de Mantenimiento

- **Mantenimiento correctivo:** Corrige errores encontrados en el sistema
- **Mantenimiento adaptativo:** Adapta el sistema a cambios en el entorno
- **Mantenimiento perfectivo:** Mejora las funcionalidades existentes
- **Mantenimiento preventivo:** Previene futuros problemas

### Documentos y Entregables
- Registro de incidencias
- Solicitudes de cambio
- Actualizaciones del sistema
- Parches de seguridad
- Historial de versiones
- Documentación actualizada del sistema
